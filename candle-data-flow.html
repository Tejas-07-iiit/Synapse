Candle Data Architecture

Technical flow of real-time market data in Synapse platform

1. External Data Source

Binance WebSocket API

Raw K-Line (Candle) data is streamed via WebSockets from Binance. This includes open, high, low, close, and volume for specific intervals (1m, 1h, etc.).

2. WebSocket Manager

services/websocket/websocketManager.ts

This service maintains the persistent connection. It listens for onMessage events, parses the JSON payload, and identifies it as a 'kline' event.

3. State Normalization (Cache)

store/dashboard/useDashboardStore.ts

The updateTicker or specialized candle actions update the Zustand store. Candles are typically stored in a Record<string, TickerInfo> map keyed by the symbol (e.g., 'BTCUSDT').

// Internal Cache Structure
tickerData: {
  "BTCUSDT": { price: 96000, high: 97000, ... },
  "ETHUSDT": { ... }
}

4. UI Sync & Rendering

components/charts/TradingViewChart.tsx

The chart component subscribes to the store. In this project, the TradingView widget handles its own internal caching once pointed at the selectedSymbol, but React components like MarketCards pull directly from the Zustand cache.
